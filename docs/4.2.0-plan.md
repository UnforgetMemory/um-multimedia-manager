# UMM 4.2.0 数据链路v4版本计划

## 一、设计原则（必须遵守）

### 1.1 Chrome MV3规范约束

**Service Worker生命周期**：
- Service Worker可能被浏览器随时终止（通常30秒无活动后）
- 每次唤醒都是全新的内存状态
- 不能使用同步API
- 没有持久化内存（内存缓存会丢失）

**存储限制**：
- IndexedDB是唯一可靠的持久化存储
- chrome.storage.local有5MB大小限制
- 内存缓存只用于临时数据（可丢失）

**并发限制**：
- 同一时间只能有一个事务
- 事务有超时限制（通常30秒）
- 没有并行查询能力

### 1.2 IndexedDB性能约束

**性能特点**：
- 没有SQL查询优化器
- 索引查询是顺序扫描
- 事务是串行的（不是并行的）
- 没有JOIN操作
- 没有部分更新（必须读取整个对象再写入）

**优化策略**：
- 合理设计索引（避免全表扫描）
- 使用游标分页（避免一次性加载所有数据）
- 批量操作（减少事务次数）
- 多级缓存（L1内存 → L2 IndexedDB）

### 1.3 数据一致性约束

**并发问题**：
- 多个Content Script可能同时读写同一数据
- 没有事务机制保证原子性
- 缓存和实际数据可能不一致
- 高并发下的竞争条件

**一致性策略**：
- 使用版本控制防止数据覆盖
- 使用乐观锁保证写入一致性
- 使用广播机制同步多Content Script状态

## 二、当前数据存储问题分析

### 2.1 问题总结

**问题1：数据存储分散**
- IndexedDB存储record（douban_records、imdb_records等）
- chrome.storage.local存储配置（settingsCache）
- 没有统一的数据管理层

**问题2：缓存机制简单**
- 读缓存只有30s/5s TTL
- 没有考虑LRU淘汰策略
- 没有考虑内存压缩
- 没有考虑Service Worker生命周期

**问题3：高峰期健壮性不足**
- 没有请求队列和限流机制
- 没有重试和降级策略
- 没有监控和告警

**问题4：站点级别缓存缺失**
- 没有为每个站点建立独立的缓存
- 缓存数据没有按站点隔离
- 缓存失效策略不完善

**问题5：IndexedDB查询性能差**
- 索引设计不合理
- 查询方式低效（全表扫描）
- 没有批量操作优化

**问题6：数据一致性问题**
- 没有版本控制
- 没有并发控制
- 缓存和实际数据可能不一致

### 2.2 性能瓶颈

| 瓶颈点 | 当前状态 | 影响 |
|--------|----------|------|
| 内存占用 | 无压缩 | 高内存使用 |
| 响应延迟 | 简单缓存 | 高峰期响应慢 |
| 并发处理 | 无队列 | 请求冲突 |
| 数据一致性 | 无版本控制 | 数据冲突 |
| 查询性能 | 全表扫描 | 大数据量查询慢 |
| 事务管理 | 简单事务 | 大事务可能失败 |

```typescript
// src/features/data-scheduler/index.ts
export class DataScheduler {
  private queue: PriorityQueue<Request>
  private rateLimiter: RateLimiter
  private retryPolicy: RetryPolicy
  private monitor: SchedulerMonitor
  
  constructor() {
    this.queue = new PriorityQueue({ maxSize: 1000 })
    this.rateLimiter = new RateLimiter({ maxPerSecond: 100 })
    this.retryPolicy = new RetryPolicy({ maxRetries: 3, backoff: 'exponential' })
    this.monitor = new SchedulerMonitor()
  }
  
  // 请求调度
  async schedule<T>(request: Request<T>): Promise<T> {
    // 1. 限流检查
    if (!this.rateLimiter.allow()) {
      throw new RateLimitError('Rate limit exceeded')
    }
    
    // 2. 入队
    const priority = this.calculatePriority(request)
    await this.queue.enqueue(request, priority)
    
    // 3. 执行
    return this.execute(request)
  }
  
  // 执行请求
  private async execute<T>(request: Request<T>): Promise<T> {
    const startTime = Date.now()
    
    try {
      // 1. 检查缓存
      const cached = await this.checkCache(request)
      if (cached) {
        this.monitor.recordCacheHit()
        return cached
      }
      
      // 2. 执行请求
      const result = await this.retryPolicy.execute(() => {
        return this.processRequest(request)
      })
      
      // 3. 更新缓存
      await this.updateCache(request, result)
      
      // 4. 记录监控
      this.monitor.recordSuccess(Date.now() - startTime)
      
      return result
    } catch (error) {
      // 5. 记录失败
      this.monitor.recordFailure(Date.now() - startTime, error)
      
      // 6. 降级处理
      return this.fallback(request, error)
    }
  }
}
```

### 3.5 多级缓存设计

```typescript
// src/features/cache-manager/index.ts
export class CacheManager {
  private l1Cache: Map<string, CacheEntry> // 内存缓存（可丢失）
  private l2Cache: IndexedDBCache // IndexedDB缓存（持久化）
  
  constructor() {
    this.l1Cache = new Map()
    this.l2Cache = new IndexedDBCache()
  }
  
  // 读取（多级缓存）
  async get<T>(key: string): Promise<T | null> {
    // 1. 检查L1内存缓存
    const l1Entry = this.l1Cache.get(key)
    if (l1Entry && !l1Entry.isExpired()) {
      return l1Entry.data as T
    }
    
    // 2. 检查L2 IndexedDB缓存
    const l2Entry = await this.l2Cache.get(key)
    if (l2Entry && !l2Entry.isExpired()) {
      // 更新L1缓存
      this.l1Cache.set(key, l2Entry)
      return l2Entry.data as T
    }
    
    // 3. 缓存未命中
    return null
  }
  
  // 写入（多级缓存）
  async set<T>(key: string, data: T, options: CacheOptions = {}): Promise<void> {
    const { ttl = 30_000, persistToDB = true } = options
    
    const entry: CacheEntry = {
      data,
      timestamp: Date.now(),
      ttl,
      version: 1
    }
    
    // 1. 写入L1内存缓存
    this.l1Cache.set(key, entry)
    
    // 2. 写入L2 IndexedDB缓存（如果需要持久化）
    if (persistToDB) {
      await this.l2Cache.set(key, entry)
    }
  }
  
  // 失效缓存
  async invalidate(key: string): Promise<void> {
    // 1. 清除L1缓存
    this.l1Cache.delete(key)
    
    // 2. 清除L2缓存
    await this.l2Cache.delete(key)
  }
  
  // 批量失效（根据模式）
  async invalidateByPattern(pattern: string): Promise<void> {
    // 1. 清除L1缓存
    for (const key of this.l1Cache.keys()) {
      if (key.startsWith(pattern)) {
        this.l1Cache.delete(key)
      }
    }
    
    // 2. 清除L2缓存
    await this.l2Cache.deleteByPattern(pattern)
  }
  
  // Service Worker生命周期管理
  async flush(): Promise<void> {
    // 将L1缓存刷新到L2缓存
    for (const [key, entry] of this.l1Cache.entries()) {
      if (entry.dirty) {
        await this.l2Cache.set(key, entry)
        entry.dirty = false
      }
    }
  }
}
```

### 3.6 内存优化层设计

```typescript
// src/features/memory-optimizer/index.ts
export class MemoryOptimizer {
  private monitor: MemoryMonitor
  private compressor: DataCompressor
  private evictionPolicy: EvictionPolicy
  
  constructor() {
    this.monitor = new MemoryMonitor()
    this.compressor = new DataCompressor()
    this.evictionPolicy = new EvictionPolicy()
  }
  
  // 内存监控（Service Worker内存有限）
  startMonitoring(): void {
    setInterval(() => {
      const usage = this.monitor.getUsage()
      
      // 如果内存使用超过阈值，触发清理
      if (usage.percentage > 70) {
        this.triggerCleanup()
      }
      
      // 记录内存使用趋势
      this.monitor.recordTrend(usage)
    }, 5000) // 每5秒检查一次
  }
  
  // 触发清理
  private triggerCleanup(): void {
    console.warn('[MemoryOptimizer] High memory usage, triggering cleanup')
    
    // 1. 清理过期缓存
    this.evictionPolicy.evictExpired()
    
    // 2. 压缩大对象
    this.compressor.compressLargeObjects()
    
    // 3. 清理最近最少使用的缓存
    this.evictionPolicy.evictLRU()
    
    // 4. 强制垃圾回收（如果可能）
    if (globalThis.gc) {
      globalThis.gc()
    }
  }
  
  // 数据压缩优化
  optimizeData(data: any): any {
    // 1. 移除null/undefined字段
    const cleaned = this.removeNullFields(data)
    
    // 2. 压缩字符串
    const compressed = this.compressStrings(cleaned)
    
    // 3. 使用更紧凑的数据格式
    return this.optimizeFormat(compressed)
  }
  
  // Service Worker生命周期管理
  async onSuspend(): Promise<void> {
    // Service Worker被终止前，清理内存
    console.log('[MemoryOptimizer] Service Worker suspending, cleaning memory')
    
    // 1. 清理所有临时缓存
    this.evictionPolicy.evictAll()
    
    // 2. 压缩所有大对象
    this.compressor.compressAll()
  }
}
```

### 3.7 安全层设计

```typescript
// src/features/security-layer/index.ts
export class SecurityLayer {
  private versionManager: VersionManager
  private optimisticLock: OptimisticLock
  private broadcast: BroadcastManager
  private auditLog: AuditLog
  
  constructor() {
    this.versionManager = new VersionManager()
    this.optimisticLock = new OptimisticLock()
    this.broadcast = new BroadcastManager()
    this.auditLog = new AuditLog()
  }
  
  // 版本控制
  async getVersion(storeName: string, key: string): Promise<number> {
    return this.versionManager.getVersion(storeName, key)
  }
  
  async nextVersion(storeName: string, key: string): Promise<number> {
    return this.versionManager.nextVersion(storeName, key)
  }
  
  // 乐观锁（写入前检查版本）
  async optimisticWrite(
    storeName: string,
    key: string,
    data: any,
    expectedVersion: number
  ): Promise<boolean> {
    // 1. 检查版本冲突
    const hasConflict = await this.optimisticLock.checkConflict(
      storeName, key, expectedVersion
    )
    
    if (hasConflict) {
      return false // 冲突，写入失败
    }
    
    // 2. 无冲突，写入数据
    const newVersion = expectedVersion + 1
    await this.optimisticLock.write(storeName, key, {
      ...data,
      version: newVersion,
      updatedAt: Date.now()
    })
    
    // 3. 广播数据变化
    await this.broadcast.change(storeName, key, newVersion)
    
    // 4. 记录审计日志
    await this.auditLog.record('write', {
      storeName,
      key,
      version: newVersion
    })
    
    return true // 成功
  }
  
  // 广播机制（多Content Script同步）
  async broadcastChange(storeName: string, key: string, version: number): Promise<void> {
    await this.broadcast.change(storeName, key, version)
  }
  
  // 订阅数据变化
  subscribe(storeName: string, callback: (data: any) => void): void {
    this.broadcast.subscribe(storeName, callback)
  }
}
```

## 五、索引设计规范

### 5.1 索引设计原则

**原则1：为常用查询字段创建索引**
- 状态查询：`status`索引
- 时间排序：`updatedAt`索引
- 复合查询：`status_updated`复合索引

**原则2：避免过多索引**
- 每个store最多5个索引
- 过多索引影响写入性能

**原则3：使用复合索引覆盖多字段查询**
- 常用查询组合：`status + updatedAt`
- 统计查询：`type + status`

### 14.1 乐观锁机制

**实现乐观锁**：
```typescript
// src/features/optimistic-lock/index.ts
export class OptimisticLock {
  private db: IDBDatabase
  
  constructor(db: IDBDatabase) {
    this.db = db
  }
  
  // 乐观锁写入
  async write<T>(
    storeName: string,
    key: string,
    data: T,
    expectedVersion: number
  ): Promise<boolean> {
    return new Promise((resolve, reject) => {
      const tx = this.db.transaction(storeName, 'readwrite')
      const store = tx.objectStore(storeName)
      
      // 读取当前数据
      const request = store.get(key)
      
      request.onsuccess = () => {
        const current = request.result
        
        // 检查版本
        if (current && current.version !== expectedVersion) {
          // 版本冲突
          resolve(false)
          return
        }
        
        // 写入新数据
        const newData = {
          ...data,
          version: expectedVersion + 1,
          updatedAt: Date.now(),
        }
        
        store.put(newData, key)
      }
      
      tx.oncomplete = () => resolve(true)
      tx.onerror = () => reject(tx.error)
    })
  }
  
  // 读取数据（获取版本号）
  async read<T>(storeName: string, key: string): Promise<{ data: T; version: number } | null> {
    return new Promise((resolve, reject) => {
      const tx = this.db.transaction(storeName, 'readonly')
      const store = tx.objectStore(storeName)
      const request = store.get(key)
      
      request.onsuccess = () => {
        const result = request.result
        if (result) {
          resolve({ data: result, version: result.version || 0 })
        } else {
          resolve(null)
        }
      }
      
      tx.onerror = () => reject(tx.error)
    })
  }
}
```

**乐观锁规范**：
- 写入前检查版本号
- 版本冲突时返回失败
- 写入时自动递增版本号
- 读取时返回版本号

### 14.2 广播机制

**实现广播机制**：
```typescript
// src/features/broadcast/index.ts
export class BroadcastManager {
  private static instance: BroadcastManager
  private subscribers: Map<string, Set<Function>>
  
  private constructor() {
    this.subscribers = new Map()
  }
  
  static getInstance(): BroadcastManager {
    if (!BroadcastManager.instance) {
      BroadcastManager.instance = new BroadcastManager()
    }
    return BroadcastManager.instance
  }
  
  // 广播数据变化
  broadcast(storeName: string, key: string, data: any): void {
    const event = `${storeName}::${key}`
    const callbacks = this.subscribers.get(event)
    
    if (callbacks) {
      for (const cb of callbacks) {
        try {
          cb(data)
        } catch (error) {
          console.error('[BroadcastManager]', error)
        }
      }
    }
  }
  
  // 订阅数据变化
  subscribe(storeName: string, key: string, callback: Function): () => void {
    const event = `${storeName}::${key}`
    
    if (!this.subscribers.has(event)) {
      this.subscribers.set(event, new Set())
    }
    
    this.subscribers.get(event)!.add(callback)
    
    return () => {
      this.subscribers.get(event)?.delete(callback)
      if (this.subscribers.get(event)?.size === 0) {
        this.subscribers.delete(event)
      }
    }
  }
  
  // 订阅整个store的变化
  subscribeStore(storeName: string, callback: Function): () => void {
    const event = `${storeName}::*`
    
    if (!this.subscribers.has(event)) {
      this.subscribers.set(event, new Set())
    }
    
    this.subscribers.get(event)!.add(callback)
    
    return () => {
      this.subscribers.get(event)?.delete(callback)
      if (this.subscribers.get(event)?.size === 0) {
        this.subscribers.delete(event)
      }
    }
  }
}
```

**广播机制规范**：
- 数据变化时广播通知
- 支持订阅单个key
- 支持订阅整个store
- 支持取消订阅

```typescript
// src/features/fallback/index.ts
export class FallbackManager {
  private static instance: FallbackManager
  private fallbacks: Map<string, Function>
  
  private constructor() {
    this.fallbacks = new Map()
  }
  
  static getInstance(): FallbackManager {
    if (!FallbackManager.instance) {
      FallbackManager.instance = new FallbackManager()
    }
    return FallbackManager.instance
  }
  
  // 注册降级函数
  register(key: string, fallback: Function): void {
    this.fallbacks.set(key, fallback)
  }
  
  // 执行降级
  async execute<T>(key: string, ...args: any[]): Promise<T> {
    const fallback = this.fallbacks.get(key)
    if (!fallback) {
      throw new Error(`No fallback registered for ${key}`)
    }
    
    return fallback(...args) as T
  }
  
  // 带降级的执行
  async withFallback<T>(
    key: string,
    primary: () => Promise<T>,
    ...args: any[]
  ): Promise<T> {
    try {
      return await primary()
    } catch (error) {
      console.warn(`[FallbackManager] Primary failed for ${key}, using fallback`, error)
      return this.execute(key, ...args)
    }
  }
}
```

**降级策略规范**：
- 核心功能必须有降级方案
- 降级方案必须可测试
- 降级时必须记录日志
- 降级后必须恢复

**实现离线支持**：

```typescript
// src/features/offline/index.ts
export class OfflineManager {
  private static instance: OfflineManager
  private isOnline: boolean
  private pendingOps: PendingOp[]
  
  private constructor() {
    this.isOnline = navigator.onLine
    this.pendingOps = []
    
    // 监听在线状态变化
    window.addEventListener('online', () => this.handleOnline())
    window.addEventListener('offline', () => this.handleOffline())
  }
  
  static getInstance(): OfflineManager {
    if (!OfflineManager.instance) {
      OfflineManager.instance = new OfflineManager()
    }
    return OfflineManager.instance
  }
  
  // 处理上线
  private handleOnline(): void {
    console.log('[OfflineManager] Back online')
    this.isOnline = true
    this.syncPendingOps()
  }
  
  // 处理离线
  private handleOffline(): void {
    console.log('[OfflineManager] Gone offline')
    this.isOnline = false
  }
  
  // 添加待同步操作
  addPendingOp(op: PendingOp): void {
    this.pendingOps.push(op)
  }
  
  // 同步待同步操作
  private async syncPendingOps(): Promise<void> {
    if (this.pendingOps.length === 0) return
    
    console.log(`[OfflineManager] Syncing ${this.pendingOps.length} pending operations`)
    
    const ops = [...this.pendingOps]
    this.pendingOps = []
    
    for (const op of ops) {
      try {
        await op.execute()
      } catch (error) {
        console.error('[OfflineManager] Failed to sync op', error)
        this.pendingOps.push(op)
      }
    }
  }
  
  // 检查是否在线
  isOnlineMode(): boolean {
    return this.isOnline
  }
}

// 待同步操作接口
interface PendingOp {
  execute: () => Promise<void>
}
```

**离线支持规范**：
- 离线时记录待同步操作
- 上线时自动同步
- 同步失败时重新加入队列
- 支持离线查询缓存

## 十六、错误恢复优化

### 16.1 错误恢复机制

**实现错误恢复机制**：
```typescript
// src/features/error-recovery/index.ts
export class ErrorRecovery {
  private static instance: ErrorRecovery
  private retryPolicy: RetryPolicy
  
  private constructor() {
    this.retryPolicy = new RetryPolicy({
      maxRetries: 3,
      backoff: 'exponential',
      baseDelay: 1000,
    })
  }
  
  static getInstance(): ErrorRecovery {
    if (!ErrorRecovery.instance) {
      ErrorRecovery.instance = new ErrorRecovery()
    }
    return ErrorRecovery.instance
  }
  
  // 带重试的执行
  async withRetry<T>(
    fn: () => Promise<T>,
    options?: RetryOptions
  ): Promise<T> {
    return this.retryPolicy.execute(fn, options)
  }
  
  // 带降级的执行
  async withFallback<T>(
    primary: () => Promise<T>,
    fallback: () => Promise<T>,
    options?: RetryOptions
  ): Promise<T> {
    try {
      return await this.retryPolicy.execute(primary, options)
    } catch (error) {
      console.warn('[ErrorRecovery] Primary failed, using fallback', error)
      return fallback()
    }
  }
  
  // 带超时的执行
  async withTimeout<T>(
    fn: () => Promise<T>,
    timeout: number
  ): Promise<T> {
    return Promise.race([
      fn(),
      new Promise<T>((_, reject) =>
        setTimeout(() => reject(new Error('Timeout')), timeout)
      ),
    ])
  }
}

// 重试策略
class RetryPolicy {
  private options: RetryOptions
  
  constructor(options: RetryOptions) {
    this.options = options
  }
  
  async execute<T>(fn: () => Promise<T>, options?: RetryOptions): Promise<T> {
    const opts = { ...this.options, ...options }
    let lastError: Error | undefined
    
    for (let i = 0; i <= opts.maxRetries; i++) {
      try {
        return await fn()
      } catch (error) {
        lastError = error as Error
        
        if (i < opts.maxRetries) {
          const delay = this.calculateDelay(i, opts)
          await this.sleep(delay)
        }
      }
    }
    
    throw lastError
  }
  
  private calculateDelay(attempt: number, options: RetryOptions): number {
    if (options.backoff === 'exponential') {
      return options.baseDelay * Math.pow(2, attempt)
    }
    
    return options.baseDelay
  }
  
  private sleep(ms: number): Promise<void> {
    return new Promise((resolve) => setTimeout(resolve, ms))
  }
}

// 重试选项接口
interface RetryOptions {
  maxRetries: number
  backoff: 'linear' | 'exponential'
  baseDelay: number
}
```

**错误恢复规范**：
- 关键操作必须有重试机制
- 重试必须有指数退避
- 重试必须有最大次数
- 重试失败时必须降级
