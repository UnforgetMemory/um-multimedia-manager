# UMM 4.0.0 基建重构计划

## 一、设计原则（必须遵守）

### 1.1 核心约束

**避免过度设计**：
- ❌ 避免大型单文件（单文件不超过300行）
- ❌ 避免层级过深（目录嵌套不超过3层）
- ❌ 避免为DDD而DDD（简单问题简单解决）

**优先DDD和子模块**：
- ✅ 优先考虑领域驱动设计（DDD）
- ✅ 优先考虑子模块化
- ✅ 但要平衡模块化和可维护性

**模块化原则**：
- ✅ 全局模块与站点模块分离
- ✅ 站点内部按访问区块分子模块
- ✅ 按注入优先级组织模块
- ✅ 站点内部共享逻辑提取到站点shared

### 1.2 平衡原则

```
简单问题 → 简单解决（不强制DDD）
复杂问题 → DDD + 子模块（但层级不超过3层）
样式问题 → 4.1.0统一重构（4.0.0简单重构）
逻辑问题 → composables + shared模块
```

## 二、当前问题总结

### 2.1 入口点混乱
- 11个入口点，职责重叠
- 3个早期脚本几乎完全相同
- 样式注入分散在多个地方

### 2.2 模块边界不清
- `content.ts` 承担过多职责
- `handlers/` 目录扁平，没有按站点分组
- `styles/` 目录没有按用途分组

### 2.3 注入优先级不明确
- 全局样式、Shadow DOM样式、页面级样式混在一起
- 没有明确的注入顺序和优先级

## 四、注入优先级设计

### 4.1 注入顺序（从高到低）
```
优先级1：页面级CSS（!important，覆盖宿主页面）
  └── douban-*-early.content → position:fixed, z-index:2147483647

优先级2：Shadow DOM样式（完全隔离）
  └── douban-*.content → style.css, breakpoints.css

优先级3：全局注入样式（需要!important）
  └── content.ts → global.ts → 搜索徽章、状态标签等

优先级4：Vue应用样式（标准CSS）
  └── popup/, options/ → style.css, typography.css
```

### 4.2 注入时机
```
document_start（立即执行）：
  └── douban-*-early → 创建Shadow DOM overlay

document_idle（DOM加载完成后）：
  ├── content.ts → 初始化、路由分发
  └── douban-*.content → 接管Shadow DOM，挂载Vue

运行时（持续运行）：
  ├── content.ts → 监听URL变化
  ├── handlers/ → 处理页面交互
  └── enhancers/ → 增强页面功能
```

## 八、WXT框架限制与文件拆分方案

### 8.1 WXT框架限制

**限制1：内容脚本必须是单文件入口**
- 每个`.content.ts`文件必须包含所有matches模式
- 无法跨内容脚本共享代码（除非使用共享模块）

**限制2：每个内容脚本独立打包**
- 每个内容脚本会被WXT打包成独立的脚本
- 无法动态加载其他内容脚本的代码

**限制3：Shadow DOM组件必须自包含**
- Shadow DOM组件必须包含所有数据提取逻辑
- 无法将提取逻辑拆分到单独文件（除非使用import）

### 8.2 当前大文件问题

| 文件 | 行数 | 问题 |
|------|------|------|
| `content.ts` | 300+ | matches数组过长，main函数过长 |
| `background.ts` | 400+ | 消息处理逻辑过长 |
| `douban-detail.content/index.ts` | 375 | 数据提取逻辑过长 |
| `global.ts` | 289 | 样式定义过长 |

### 8.5 拆分原则

1. **单一职责**：每个文件只负责一个功能
2. **按功能分组**：相关功能放在同一目录
3. **按类型分组**：相同类型的文件放在一起
4. **避免循环依赖**：模块之间不要有循环引用
5. **保持接口清晰**：每个模块都有明确的导出接口

## 九、成功标准

1. **模块清晰**：每个模块职责单一，边界清晰
2. **注入有序**：注入优先级明确，顺序合理
3. **代码复用**：重复代码减少50%以上
4. **维护简单**：修改一处逻辑，所有引用点自动更新
5. **扩展容易**：添加新平台只需实现接口，无需修改核心代码
6. **文件拆分**：单文件不超过300行，大文件按功能拆分

```typescript
// src/shared/state/index.ts
export class StateManager {
  private state: Map<string, any>
  private subscribers: Map<string, Set<Function>>
  
  constructor() {
    this.state = new Map()
    this.subscribers = new Map()
  }
  
  // 获取状态
  get<T>(key: string): T | undefined {
    return this.state.get(key) as T
  }
  
  // 设置状态
  set<T>(key: string, value: T): void {
    this.state.set(key, value)
    this.notify(key, value)
  }
  
  // 订阅状态变化
  subscribe(key: string, callback: Function): () => void {
    if (!this.subscribers.has(key)) {
      this.subscribers.set(key, new Set())
    }
    this.subscribers.get(key)!.add(callback)
    
    return () => {
      this.subscribers.get(key)?.delete(callback)
    }
  }
  
  // 通知订阅者
  private notify(key: string, value: any): void {
    const callbacks = this.subscribers.get(key)
    if (callbacks) {
      for (const cb of callbacks) {
        try { cb(value) } catch (e) { console.error('[StateManager]', e) }
      }
    }
  }
}
```

**状态管理规范**：
- 主题状态统一管理
- 用户设置统一管理
- 页面状态统一管理
- 缓存状态统一管理

```typescript
// src/shared/utils/dom.ts
export class DomUtils {
  // 统一的DOM查询
  static query<T extends Element>(selector: string, parent: Element = document): T | null {
    return parent.querySelector(selector) as T
  }
  
  // 统一的DOM查询（多个）
  static queryAll<T extends Element>(selector: string, parent: Element = document): T[] {
    return Array.from(parent.querySelectorAll(selector)) as T[]
  }
  
  // 统一的等待元素
  static waitForElement(selector: string, timeout: number = 5000): Promise<Element> {
    return new Promise((resolve, reject) => {
      const element = document.querySelector(selector)
      if (element) {
        resolve(element)
        return
      }
      
      const observer = new MutationObserver(() => {
        const element = document.querySelector(selector)
        if (element) {
          observer.disconnect()
          resolve(element)
        }
      })
      
      observer.observe(document.body, { childList: true, subtree: true })
      
      setTimeout(() => {
        observer.disconnect()
        reject(new Error(`Element ${selector} not found within ${timeout}ms`))
      }, timeout)
    })
  }
  
  // 统一的事件监听
  static on(element: Element, event: string, handler: Function, options?: AddEventListenerOptions): () => void {
    element.addEventListener(event, handler as EventListener, options)
    return () => element.removeEventListener(event, handler as EventListener, options)
  }
}
```
